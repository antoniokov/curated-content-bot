### End-of-session checklist

When I say "we're done", "ship it", "finish up", or similar — follow this sequence:
1. If you haven't yet, run tests and confirm they all pass. If any fail, fix them before proceeding.
2. Update plan.md: mark the current task as completed and move it to the bottom of "Done" section. If there's no task for what we've accomplished, add a new task to the "Done" section marked as done.
3. Update README.md if it requires updating and you haven't done it yet.
4. Stage all changes related to this session.
5. Propose a short meaningful commit message (one line, imperative mood). Show it to me and wait for my approval or edits before committing.
6. After I confirm, commit and push to the current branch without asking for further permissions.
